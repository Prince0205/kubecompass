ISSUE #2: NAMESPACE DROPDOWN NOT UPDATING INSTANTLY
================================================================================

PROBLEM:  Selecting namespace required multiple browser reloads to see changes
ROOT CAUSES:
  1. Broken dependency tracking in useApi hook (JSON.stringify logic unreliable)
  2. Spread operator [...dependencies] in useEffect unpredictable
  3. Race condition: API called before backend namespace updated
  4. Missing activeNamespace in direct dependencies

FIXES APPLIED:

Fix #1 - Simplified useApi.js Dependency Tracking
  File: ui/src/hooks/useApi.js
  Changes:
    ❌ Removed: prevDependencies ref with JSON.stringify comparison
    ❌ Removed: Spread operator [...dependencies] in useEffect
    ✅ Added: Direct activeCluster and activeNamespace in dependencies
    ✅ Result: React's built-in comparison now works reliably

  Old Dependency Array:  [fetchData, cacheKey, ...dependencies]
  New Dependency Array:  [fetchData, cacheKey, activeCluster, activeNamespace]

Fix #2 - Made selectNamespace Async
  File: ui/src/context/AppContext.jsx
  Changes:
    ✅ Changed selectNamespace from sync to async
    ✅ Added await to axios.post
    ✅ Ensures backend namespace updated before components fetch

  Old: axios.post(...).catch(...)  // Fire and forget
  New: await axios.post(...)       // Wait for completion

Fix #3 - Cache Clearing in Dashboard
  File: ui/src/pages/Dashboard.jsx
  Changes:
    ✅ Added cache clearing on metrics fetch
    ✅ Ensures fresh data when namespace changes

RESULT: ✅ Namespace changes now update resources INSTANTLY (no reload needed)
