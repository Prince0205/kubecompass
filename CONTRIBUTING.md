# Contributing to Kubernetes Compass

Thanks for your interest! A few guidelines to get started:

- Fork the repo and create small, focused pull requests.
- Write tests for new functionality and run existing tests locally.
- Keep sensitive information (kubeconfigs, credentials) out of commits.
- Follow the existing code style (PEP8 for Python) and run linters in CI.

If you want to propose major architecture changes, open an issue first to discuss.
